Welcome to the API

Direct access to the endpoints is restricted and a valid token is required to access the API. Please use an API development ecosystem (ADE) such as hoppscotch to access enpoints.

Endpoints:

Users

This is required to generate a token for authentication and authourisation, please do this before accessing the tasks endpoint.
Only tasks create by you will be shown!

  • Login with existing account: https://protectthepanda-api.onrender.com/users/login

    • In the ADE send a POST request with a json in the following format:
     {"email": "YOUR-EMAIL-ADDRESS",
     "password": "YOUR-PASSWORD"}

  • Create a new account account:https://protectthepanda-api.onrender.com/users/signup

    • In the ADE send a POST request with a json in the following format:
     {"email": "YOUR-EMAIL-ADDRESS",
     "password": "YOUR-PASSWORD"}

A token will be generated please copy this token for accessing tasks.

Tasks

Paste the token generated in to the ADE authentication section as a "token" to access tasks endpoints.
Access will not be granted without a valid token.

  • Get all tasks: https://protectthepanda-api.onrender.com/tasks/

    • In the ADE send a GET request

  • Get one task by ID: https://protectthepanda-api.onrender.com/tasks/:id

    • In the ADE send a GET request

  • Post a new task: https://protectthepanda-api.onrender.com/tasks/

    • In the ADE send a POST request with a json in the following format:
     {"task": "YOUR-NEW-TASK",
     "time": SET-DURATION-FOR-TASK}

  • Delete an existing task: https://protectthepanda-api.onrender.com/tasks/:id

    • In the ADE send a DELETE request with an empty json body.

  • Mark a task a completed: https://protectthepanda-api.onrender.com/tasks/:id/complete

    • In the ADE send a PATCH request with an empty json body.
    When the task is fetched again it will be marked a completed.

  • Modify an existing task: https://protectthepanda-api.onrender.com/tasks/:id

    • In the ADE send a PATCH request with a json in the following format:
     {"task": "MODIFIED-TASK-NAME",
     "time": MODIFIED-TASK-DURATION}
    When the task is fetched again it will show the modified task.
